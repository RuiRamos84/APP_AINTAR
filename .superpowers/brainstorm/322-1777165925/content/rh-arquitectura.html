Arquitectura Completa — Módulo RH Pessoal

Schema de BD, estrutura frontend e fluxos de aprovação. Revê e confirma antes de avançarmos para o plano de implementação.

🗄️ Base de Dados — Tabelas & Views

TABELAS (tb_ / ts_)
ts_rh_config — saldo por colaborador/ano
pk · tb_user_fk · year · dias_total · dias_gozados · created_at
tb_rh_ferias — pedidos de férias
pk · tb_user_fk · data_inicio · data_fim · dias_uteis · tt_tipo_fk · ts_estado_fk · notas · created_at
tb_rh_ferias_workflow — aprovações
pk · tb_rh_ferias_fk · step (1=superior/2=rh) · tb_user_fk · estado · notas · ts
tb_rh_faltas — registos de faltas
pk · tb_user_fk · data · tt_tipo_falta_fk · ts_estado_fk · justificativo_path · notas · created_at
tb_rh_faltas_workflow — aprovações faltas
pk · tb_rh_faltas_fk · step · tb_user_fk · estado · notas · ts
ts_rh_horario — horário fixo
pk · tb_user_fk · descr · hora_entrada · hora_saida · dias_semana · data_inicio · data_fim
tb_rh_piquete_escala — escala semanal
pk · tb_user_fk · data_inicio · data_fim · confirmado · ts_confirmacao · estado
tb_rh_piquete_ocorrencia — log de ocorrências
pk · tb_piquete_escala_fk · tt_tipo_fk · descr · equipas_acionadas · evidencia_path · created_by · ts
ts_feriados — feriados PT (pré-carregado)
pk · data · descr · nacional · regiao
LOOKUPS (tt_)
tt_rh_tipo_ferias — Férias, Tolerância, Aniversário
tt_rh_tipo_falta — Justificada, Injustificada, Tolerância, Licença
tt_rh_estado — Pendente, Validado, Aprovado, Rejeitado
tt_rh_piquete_ocorrencia — Chamada, Intervenção, Equipa
VIEWS LEITURA (vbl_)
vbl_rh_colaborador — ts_client + ts_rh_config + horário
vbl_rh_ferias — pedidos + estado texto + user info
vbl_rh_faltas — faltas + tipo + estado + user info
vbl_rh_horario — horário + user info
vbl_rh_piquete — escala + confirmação + user info
vbl_rh_piquete_ocorrencias — log detalhado
FUNÇÕES (fbo_ / fn_)
fbo_rh_ferias(pop, pk, ...) — INSERT/UPDATE férias
fbo_rh_ferias_workflow(pk, step, estado, ...) — aprovar/rejeitar
fbo_rh_faltas(pop, pk, ...) — INSERT/UPDATE faltas
fbo_rh_faltas_workflow(pk, step, estado, ...)
fbo_rh_horario(pop, pk, ...) — gerir horários
fbo_rh_piquete_generate(year, month) — gerar escala
fbo_rh_piquete_confirmar(pk, user_fk)
fbo_rh_ocorrencia(pop, pk, ...)
fn_rh_dias_uteis(inicio, fim) — conta dias úteis excluindo feriados PT

🔄 Fluxos de Aprovação

FÉRIAS
1. Colaborador

↓
2. Superior
Valida / Rejeita
↓
3. Admin RH
Aprovação final
↓
Saldo debitado automaticamente
FALTAS
1. Colaborador/Superior
Comunica falta
↓
2. Colaborador
Upload justificativo
↓
3. Superior
Valida
↓
4. Admin RH
Aprovação final
